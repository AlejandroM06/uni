GIT
To remove the file from both the repository and your local filesystem: git rm <filename>
Commit the Deletion: Commit the changes to your local repository: git commit -m "Deleted <filename>"
Push Changes to Remote: Push the committed changes to your GitHub repository: git push origin <branch-name> (master)

1. Stage Changes:
After modifying files in your local working directory, you need to tell Git which of these changes you want to include in your next commit. This is called "staging." = git add <file_name> # To stage a specific file / git add . # To stage all changes in the current directory
2. Commit Changes:
Once changes are staged, you can commit them to your local Git repository. A commit is a snapshot of your project at a specific point in time. = git commit -m "Your descriptive commit message here"
3. Push Changes to GitHub:
After committing locally, you need to push these committed changes to your remote GitHub repository to make them available online. = git push origin <your_branch_name> (master)
